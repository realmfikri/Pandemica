Help

The transmission modifier multiplies the base infection probability in the simulation. A value of 1.0 keeps the default behavior, while lower numbers dampen spread and higher numbers (up to 1.0) raise the likelihood that an encounter results in infection.

Adjust the slider to send a ControlUpdate to the server. Everyone connected to the simulation will see the current rate stay in sync.

Use the lockdown switch to freeze movement: the backend sets a global speed modifier to a tenth of normal and broadcasts the updated state so all clients animate consistently.

The capacity controls set how many active infections hospitals can absorb. When infected exceeds capacity, the simulation amplifies per-tick death probability by the overload multiplier. Lower multipliers keep the system resilient; higher values model severe strain.
